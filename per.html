Usando o schtasks para persistência no Windows

Observação: Para utilizar esta técnica, é necessário ter acesso de administrador.

O schtasks é uma ferramenta de linha de comando do Windows que permite gerenciar tarefas agendadas usando a Task Scheduler. Essa ferramenta pode ser usada para criar, modificar, excluir e executar tarefas agendadas. Neste artigo, discutiremos como usar o schtasks para obter persistência no Windows.

Introdução ao schtasks

O schtasks é uma ferramenta de linha de comando que permite gerenciar tarefas agendadas na Task Scheduler do Windows. Essa ferramenta pode ser usada para criar tarefas que executam scripts ou aplicativos maliciosos, permitindo que um atacante obtenha persistência no sistema comprometido.

Criando uma tarefa usando o schtasks

Para criar uma tarefa usando o schtasks, você pode usar o seguinte comando:

schtasks /create /tn "Nome da Tarefa" /tr "Caminho do Script ou Aplicativo" /sc minute /mo 5

Neste comando, /create é usado para criar uma nova tarefa, /tn é usado para definir o nome da tarefa, /tr é usado para especificar o caminho do script ou aplicativo que será executado, /sc é usado para definir a frequência da tarefa (minuto, hora, dia, semana ou mês) e /mo é usado para especificar a frequência em minutos.

Modificando uma tarefa usando o schtasks

Para modificar uma tarefa usando o schtasks, você pode usar o seguinte comando:

schtasks /change /tn "Nome da Tarefa" /tr "Novo Caminho do Script ou Aplicativo" /sc minute /mo 10

Neste comando, /change é usado para modificar uma tarefa existente, /tn é usado para especificar o nome da tarefa que será modificada, /tr é usado para especificar o novo caminho do script ou aplicativo que será executado, /sc é usado para definir a nova frequência da tarefa e /mo é usado para especificar a nova frequência em minutos.

Excluindo uma tarefa usando o schtasks

Para excluir uma tarefa usando o schtasks, você pode usar o seguinte comando:

schtasks /delete /tn "Nome da Tarefa"

Neste comando, /delete é usado para excluir uma tarefa existente e /tn é usado para especificar o nome da tarefa que será excluída.
